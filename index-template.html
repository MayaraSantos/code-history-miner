Why?

There is a lot of interesting and useful data captured in version control systems, yet we don't use it that much. This is an attempt to make looking at project history easier.

Find out what happened on project. Find trends in code changes. Look for code-change/behavioral patterns.

Features

Similar to continuous integration servers Code History Miner is a web server to run inside your company. You can setup multiple projects pointing them to VCS repositories. For each project you will get daily updated visualizations of project code history. Analysis works on file level and, therefore, is programming language-agnostic.
See visualization examples for JUnit, Cucumber, Ruby on Rails, jQuery, Clojure, Groovy, Scala, IntelliJ.


Requirements

Any OS with Java >= 1.6. Git/Svn/Mercurial. Browser with SVG support. Project size < 1M LOC.
